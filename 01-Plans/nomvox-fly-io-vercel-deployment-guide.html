🚀 NomVox Deployment Guide

Fly.io (Go backend) + Vercel (Next.js frontend) · IBM SkillsBuild AI Builders Challenge July 2026

Architecture Overview

🪰 Fly.io — Go API Server

  • 📦 Go 1.22 binary · nomvox-api app
  • 🔑 Holds secret env vars (API keys)
  • 🌍 Region: iad (US East, changeable)
  • 💤 Auto-stop when idle (free tier friendly)

▲ Vercel — Next.js Frontend

  • ⚛️ Next.js 16 · React 19 · Tailwind v4
  • 🔧 One env var: NEXT_PUBLIC_API_URL
  • 🆓 Free Hobby plan sufficient
  • 🔗 Points to Fly.io backend URL

All Environment Variables — What They Are & Where To Find Them

VariableRequired?Where to find it
WATSONX_API_KEY Required cloud.ibm.com/iam/apikeys → "Create an IBM Cloud API key" → Copy the key immediately (shown once).
Must have watsonx.ai service access. The key is ~44 alphanumeric chars.
WATSONX_PROJECT_ID Required dataplatform.cloud.ibm.com → Open your watsonx project → Settings tab → Copy "Project ID" (UUID format: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx)
WATSONX_URL Required Your watsonx region endpoint. Usually https://us-south.ml.cloud.ibm.com or https://ca-tor.ml.cloud.ibm.com. Check your watsonx project's "API" tab for the correct regional URL.
GOOGLE_AI_API_KEY Required aistudio.google.com/app/apikey → "Create API key" → Select a Google Cloud project → Copy key.
⚠️ Note: Image generation (Gemini Flash Image) requires billing credits. Text/SVG generation works on the free tier.
REDIS_URL Required upstash.com → Create a Redis database → "Connect" tab → Copy the UPSTASH_REDIS_URL (starts with rediss://default:…).
Free plan: 10,000 commands/day — sufficient for demo.
VERCEL_BLOB_TOKEN Optional This is for Vercel Blob storage (image uploads) — NomVox does not currently use it.
If you see it in .env.example, it was added for future use. You do NOT need it for deployment.
If needed later: Vercel Dashboard → Your Project → Storage tab → "Connect Store" → Create Blob store → the token appears as BLOB_READ_WRITE_TOKEN in env vars.
ALLOWED_ORIGINS Required on Fly Your Vercel deployment URL, e.g. https://nomvox.vercel.app. Prevents CORS errors. Set this after you know the Vercel URL (Step 10).
NEXT_PUBLIC_API_URL Required on Vercel Your Fly.io backend URL, e.g. https://nomvox-api.fly.dev. Set this in Vercel project settings (Step 9).

Part 1 — Deploy Go Backend to Fly.io

Step 1 — Add a Dockerfile (if not present)

Check: a Dockerfile should already exist at the project root. If not, create it:

FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN  go mod download
COPY . .
RUN  go build -o nomvox-server ./cmd/server

FROM alpine:3.20
WORKDIR /app
COPY --from=builder /app/nomvox-server .
EXPOSE 8080
CMD  ["./nomvox-server"]

Step 2 — Create the Fly.io app (first time only)

# Run from the repo root (where fly.toml is)
fly apps create nomvox-api   # creates the app on Fly.io

The fly.toml already has app = "nomvox-api" — no changes needed.

Step 3 — Set all secret environment variables

fly secrets set \
  WATSONX_API_KEY="your-44-char-ibm-api-key" \
  WATSONX_PROJECT_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx" \
  WATSONX_URL="https://ca-tor.ml.cloud.ibm.com" \
  GOOGLE_AI_API_KEY="your-google-api-key" \
  REDIS_URL="rediss://default:your-token@your-host.upstash.io:6379" \
  ALLOWED_ORIGINS="https://nomvox.vercel.app"
# Note: ALLOWED_ORIGINS will be updated after Vercel deploy (Step 10)
✓ Secrets are encrypted — never appear in logs or code. Verify with: fly secrets list

Step 4 — Deploy to Fly.io

# Make sure Docker Desktop is running first!
fly deploy
# First deploy takes 3-5 minutes (builds Docker image, uploads)
# Subsequent deploys take ~60 seconds

After deploy, your backend is live at: https://nomvox-api.fly.dev

Step 5 — Test the backend

# Health check
Invoke-RestMethod https://nomvox-api.fly.dev/api/ping

# Expected: {"status":"ok","service":"nomvox"}

# Full visual debug (tests watsonx SVG logos + Gemini)
Invoke-RestMethod https://nomvox-api.fly.dev/api/debug-visuals | ConvertTo-Json

Part 2 — Deploy Next.js Frontend to Vercel

Step 6 — Push to GitHub (required for Vercel)

git add -A
git commit -m "feat: production deploy"
git push origin main
⚠️ Make sure .env is in .gitignore — never commit real API keys!

Step 7 — Create Vercel project (first time only)

A
Go to vercel.com/new
B
Click "Import" next to your NomVoxBranding GitHub repo
C
IMPORTANT: Set the Root Directory to frontend
Vercel must point at the frontend/ subdirectory, not the repo root.
D
Framework preset: auto-detected as Next.js ✓
E
Build Command: npm run build (auto-filled) ✓

Step 8 — Add the NEXT_PUBLIC_API_URL environment variable

Before clicking Deploy, in the "Environment Variables" section add:

NameValueEnvironment
NEXT_PUBLIC_API_URLhttps://nomvox-api.fly.devProduction, Preview, Development
✓ This is the only env var the frontend needs — it tells Next.js where to call the Go API.

Then click Deploy. First deploy takes ~2 minutes.

Step 9 — Get your Vercel URL and update Fly.io CORS

After deploy, Vercel shows your URL, e.g. https://nomvox-abc123.vercel.app

A
Optional: add a custom domain in Vercel → Domains → Add. Otherwise use the auto-generated URL.
B
Update Fly.io CORS to allow your Vercel domain: 
fly secrets set ALLOWED_ORIGINS="https://nomvox-abc123.vercel.app"
# If you have multiple domains:
fly secrets set ALLOWED_ORIGINS="https://nomvox.vercel.app,https://nomvox.app"

Step 10 — Verify end-to-end

✓
Open your Vercel URL in a browser
✓
Type a brand idea (e.g. "eco coffee for young professionals") and hit Generate
✓
Names + availability check should return in ~20 seconds
✓
Click a name → Visual Identity should load with SVG logos + landing page mockup

Troubleshooting

SymptomFix
CORS error in browser consoleRun fly secrets set ALLOWED_ORIGINS="https://your-vercel-url.vercel.app" and redeploy
Names not generating (504 timeout)watsonx is slow on first call. Check fly logs — look for WATSONX_API_KEY error. Also confirm WATSONX_PROJECT_ID is correct UUID.
Logos show CSS placeholders onlyNormal on free tier — SVG logos via watsonx should still load (~30s). Check /api/debug-visuals for svg_logo_ok: true.
Vercel build fails "Cannot find module"Ensure Root Directory is set to frontend in Vercel project settings.
fly deploy fails "no Dockerfile"Check Dockerfile exists at repo root. Alternatively add [build] dockerfile = "Dockerfile" to fly.toml.
Redis errors in logsConfirm REDIS_URL starts with rediss:// (TLS, double-s). Upstash free tier requires TLS.
Landing page mockup is black/no colourThis is a new fix in current build. Rebuild nomvox-server.exe with go build -o nomvox-server.exe ./cmd/server and redeploy.

Quick Re-deploy Commands (after code changes)

# Backend (Go) — run from repo root
fly deploy

# Frontend (Next.js) — push to GitHub, Vercel auto-deploys
git push origin main

# OR force manual Vercel deploy
cd frontend
vercel --prod

About VERCEL_BLOB_TOKEN

You do NOT need VERCEL_BLOB_TOKEN for NomVox to work.

It appears in .env.example as a placeholder for a planned future feature (uploading reference images to Vercel Blob storage for the vision analysis feature).

If you need it in the future:
1. Vercel Dashboard → Your Project → Storage tab (left sidebar)
2. Click "Create Database" → Select "Blob"
3. Give it a name → Create
4. The token BLOB_READ_WRITE_TOKEN is automatically added to your project's env vars
5. Copy it to your .env as VERCEL_BLOB_TOKEN=…
