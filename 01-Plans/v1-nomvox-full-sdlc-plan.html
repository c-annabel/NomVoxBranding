1 · Problem Statement

Every new business, creator, or side-hustle founder faces the same blank-page problem: what do I call this, and does anyone already own that name? Today that means juggling a name-brainstorm session, a domain registrar tab, separate searches on Instagram/X/TikTok, a freelancer quote for logo concepts, and a week of back-and-forth before you have anything you can confidently share.

The gap is not lack of ideas — it is the iteration loop. Current tools (Namelix, Namechk, Looka) are one-shot generators with no conversational memory. A user cannot say "I liked the energy of option 3 but want it shorter and less corporate" and have the tool genuinely respond to that signal.

2 · Solution Overview

NomVox is a conversational AI brand-identity partner. A founder describes their idea in plain language; NomVox generates names, taglines, visual mood boards, a quick logo concept, and a hero-section landing-page mockup — all within a single iterative session that remembers what the user liked or rejected and refines accordingly.

4 · Gap Analysis & Solutions

GAP 1 — Zero all-green results
No name in a batch may be fully available. Solution: Show a priority matrix — rows are names, columns are platforms. User toggles which platforms matter. Show "best fit" score (e.g., domain + Instagram available = 4/5). Offer a "closest match" row with minor spelling variants that unlock availability.
GAP 2 — Regenerate memory
LLM must not repeat rejected names or lose liked names between calls. Solution: Maintain a session object in Redis: liked[], rejected[], direction_notes[]. Every /api/generate call injects this as a system-prompt prefix. Names in rejected[] are explicitly excluded.
GAP 3 — Mockup scope creep
Full site mockup is weeks of work. Solution: Scope to a single hero-section HTML mock (headline, sub-headline, CTA button, brand color palette, placeholder logo). Generated by Granite as HTML/CSS + image slot filled by image-gen. Render in an <iframe> sandbox.
GAP 4 — Handle availability accuracy
Direct HTTP probes can be blocked or return false positives. Solution: Probe Instagram, X, TikTok, Threads, YouTube via URL HEAD request + HTTP status heuristic (200 = taken, 404 = available). For domains, use a proper RDAP/WHOIS API (Domainr or similar). Cache results for 10 min. Show confidence label.
GAP 5 — Screenshot of taken handles
Show who owns a taken handle. Solution: Use Puppeteer (server-side) or Screenshotone API to capture a viewport-crop of the profile page when a handle is marked taken. Store screenshot as a signed URL. Show thumbnail with "Currently owned by" overlay. Rate-limit to avoid blocks.
GAP 6 — Logo generation quality
Prompt-to-logo quality varies. Solution: Generate 3 logo concept variants (minimalist icon, wordmark, emblem). Auto-extract brand colors from the mood board using a color quantization step. Provide a "regenerate this one" button per variant without regenerating others.
GAP 7 — Cold-start blank page
Users don't know what to type. Solution: Structured prompt builder with guided fields (industry dropdown, brand adjectives chips, target audience, style preference, color mood). All fields optional — each filled field sharpens the prompt. Show "Inspire me" examples.
GAP 8 — Judging demo readiness
3-minute video must show the full loop. Solution: Seed a "happy path" demo with a pre-seeded brand brief (e.g., "eco coffee for young professionals"). Record: input → names → availability → react → visuals → export. All in under 2:30 to leave 30s for architecture slide.

Availability
Domain & Handle Availability Engine

  • Status heuristic: 200 = taken, 404 = available, 429/5xx = unknown (show "check manually" badge)
  • Results cached in Redis for 10 min to avoid repeated probes during same session
  • Priority matrix UI: rows = name candidates, columns = platforms; user toggles priority
  • Closest-variant fallback: if no name is fully green, append numeric suffix or phonetic swap variants and re-check

8 · IBM Granite Prompt Design

System prompt (injected every call)

SectionContent
RoleYou are NomVox, an expert brand-naming partner. You generate creative, memorable brand names and taglines aligned with the user's brand direction.
ConstraintsNever suggest names from this rejected list: {rejected[]}. The user has liked these names/styles: {liked[]}. Previous direction notes: {direction_notes[]}. Do not repeat any name from: {allGenerated[]}.
Output formatReturn a JSON array of 5–10 NameCard objects: { name, tagline, toneReasoning, styleTags[], shortDescription, longDescription }.
Quality rulesNames should be ≤ 12 characters, pronounceable, and globally inoffensive. Taglines ≤ 8 words. Reasoning should be 1–2 sentences explaining the brand tone.

Image-gen prompt template (mood board)

FieldExample value
Style directive"editorial product photography, flat lay, {style_adjectives}"
Color anchors"dominant colors: {palette_colors}, cohesive palette"
Subject"{industry} brand imagery, {target_audience} lifestyle"
Negative prompt"text, logos, watermarks, low resolution, busy composition"
Aspect ratio1:1 (square), 512×512, seed randomized per variant
